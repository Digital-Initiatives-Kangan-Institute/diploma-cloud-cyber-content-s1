# Topic 10 — Documenting the build, handover & sign-off · Coverage

**Topic 10 of 10** · **AT2 content Topic** (documentation + closure) · teaching source: bespoke.

This file is the **coverage spec** for the Topic.

**Depth ceiling: SKILL.** The technical build is Topics 6–9; this Topic teaches documenting it, handing it over and closing it out. Comes last in AT2, ahead of the Deployment Report submission.

---

## What this Topic must cover

Closing out the build: user documentation, the Deployment Report, sign-off, and the written knowledge evidence. Four components:

- **C1 — IaC user documentation.** Create user documentation including the IaC templates — how to deploy, update and delete the stacks with parameters — in a clear, logical format.
- **C2 — The Deployment Report & build feedback.** Assemble the report (build narrative, configuration decisions, test/troubleshooting evidence); confirm, seek and respond to feedback with the required personnel.
- **C3 — Final sign-off.** Obtain final sign-off from the required personnel — the build closure / approval to proceed to production.
- **C4 — Written knowledge evidence.** Contextual written responses on the student's own build — IaC concepts, testing/debugging, and the shared cloud foundations — assembled into the report's KE appendix.
